# キーワードリサーチスキル

ユーザーが入力したブログ記事テーマについて、Web検索を活用してキーワード・共起語・関連用語を調査し、検索需要を予測するレポートを生成する。

## ワークフロー

以下の手順で調査を実行すること。手順を省略しないこと。

### Step 1: テーマの確認と検索クエリの設計

ユーザーが入力したテーマから以下を特定する：
- メインキーワード（1〜2語）
- 想定ターゲット（初心者/中級者/上級者）
- 掲載先サイト候補（chromebookguide.net / pcefan.com / aiwriters.jp 等）

### Step 2: Web検索の実行（最低5回）

以下のパターンでweb_searchを実行する。検索クエリは日本語で行うこと。

1. **メインキーワード検索**: `{メインキーワード}`
   - 上位表示サイトの傾向を把握
2. **サジェスト調査**: `{メインキーワード} おすすめ` / `{メインキーワード} 比較` / `{メインキーワード} 使い方`
   - ユーザーの検索意図を把握
3. **共起語調査**: `{メインキーワード} とは` / `{メインキーワード} メリット デメリット`
   - 記事に含めるべき関連ワードを収集
4. **競合調査**: `{メインキーワード} レビュー` / `{メインキーワード} ブログ`
   - 上位記事の構成・キーワード傾向を分析
5. **ニーズ深堀り**: `{メインキーワード} 悩み` / `{メインキーワード} できない` / `{メインキーワード} 選び方`
   - 潜在ニーズ・ロングテールキーワードを発見

テーマによっては追加で以下も実行する：
- 英語キーワード検索（海外トレンドの調査）
- 最新ニュース検索（`{キーワード} 2025` `{キーワード} 2026` `{キーワード} 最新`）
- 価格・セール系検索（製品レビュー系の場合）

### Step 3: 検索結果の分析

Web検索結果から以下を抽出・整理する：

- **上位10サイトの傾向**: タイトル・見出しに頻出する語句
- **ユーザーの検索意図の分類**: 情報収集型 / 比較検討型 / 購入型 / 問題解決型
- **共起語候補**: 検索結果に繰り返し登場する関連ワード
- **コンテンツギャップ**: 上位記事が扱っていないが需要がありそうなトピック

### Step 4: 需要予測

以下の判定基準で検索需要を予測する：

| 需要レベル | 判定基準 |
|-----------|---------|
| 🔥 高需要 | 大手メディア・企業サイトが上位を占める。広告出稿あり。サジェストが多い |
| 📈 中需要 | 個人ブログ〜中規模メディアが上位。サジェスト数は普通 |
| 📊 低需要（ニッチ） | 個人ブログが上位で競合少ない。サジェストが少ない |
| 🆕 新興需要 | 検索結果が少ないが、SNSやニュースでの言及が増加中 |

需要予測には以下の観点も加味する：
- 季節性（セール時期、新学期、年末年始等）
- トレンド性（新製品発表、アップデート等）
- 常緑性（年間を通じて安定したアクセスが見込めるか）

- 「{KW} vs {競合}」→ 他製品との違い

### 問題解決フェーズ

- 内部リンク戦略（既存記事との連携）

## 重要な注意事項

- Web検索は必ず実行すること。検索せずに推測だけでレポートを作成しない
- 検索結果の情報はパラフレーズして記載する（著作権遵守）
- 需要予測は検索結果の傾向からの推定であり、正確な検索ボリューム数値ではないことをレポートに明記する
- テーマがChromebook関連ならchromebookguide.net、ガジェット全般ならpcefan.com、AI関連ならaiwriters.jpを掲載先として推奨する
- 調査結果はMarkdownファイルとして /mnt/user-data/outputs/ に保存し、ユーザーに提供する
